3.2.1.1 Template

Combinatorial logic is described with a generator function code template as follows: 

def combinatorialLogic(<arguments>)
    while 1:
        yield <input signals>
        <functional code>

The overall code is wrapped in a while 1 statement to keep the generator alive. All input signals are clauses in the yield statement, so that the generator resumes whenever one of the inputs changes.

Note that the input signals are listed explicitly in the template. Alternatively, MyHDL provides the always_comb() function to infer the input signals automatically. 3.1. The argument of always_comb() is a local function that specifies what happens when one of the input signals changes. It returns a generator that is sensitive to all inputs, and that will run the function argument whenever an input changes. Using always_comb, the template can be rewritten as follows: 

def combinatorialLogic(<arguments>)
    def logicFunction():
        <functional code>
    return always_comb(logicFunction)
